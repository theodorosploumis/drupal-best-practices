# AI Commands for Drupal Best Practices

Use these slash commands in compatible AI-enabled CLIs or IDEs to load context-specific guidance.

- `/drupal-best-practices-nodes` → Nodes (bundle naming, revisions, view modes)
- `/drupal-best-practices-blocks` → Blocks (custom block types/plugins, machine names)
- `/drupal-best-practices-taxonomy` → Taxonomy (vocab usage vs. list fields)
- `/drupal-best-practices-other-entities` → Other content entities (paragraphs, media, image styles)
- `/drupal-best-practices-fields` → Fields (naming patterns, reuse guidance, directories)
- `/drupal-best-practices-views` → Views (machine names, displays, access, formatting)
- `/drupal-best-practices-forms` → Forms (Webform vs. Contact)
- `/drupal-best-practices-text-formats` → Text formats and editors (CKEditor usage, allowed tags)
- `/drupal-best-practices-users` → Users, roles & permissions (persona roles, permissions, dashboards)
- `/drupal-best-practices-theming` → Theming & templates (theme naming, atomic design, twig/preprocess)

**Usage tip:** Pair commands with file or module names (e.g., `/drupal-best-practices-views blog_listing`) to request targeted checks or generation.
