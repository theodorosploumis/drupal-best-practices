# AI Commands for Drupal Best Practices

Use these slash commands in compatible AI-enabled CLIs or IDEs to load context-specific guidance. Each command maps to README sections 2 or 3.

- `/drupal-best-practices-nodes` → 2.1 Nodes (bundle naming, revisions, view modes)
- `/drupal-best-practices-blocks` → 2.2 Blocks (custom block types/plugins, machine names)
- `/drupal-best-practices-taxonomy` → 2.3 Taxonomy (vocab usage vs. list fields)
- `/drupal-best-practices-other-entities` → 2.4 Other content entities (paragraphs, media, image styles)
- `/drupal-best-practices-fields` → 2.5 Fields (naming patterns, reuse guidance, directories)
- `/drupal-best-practices-views` → 2.6 Views (machine names, displays, access, formatting)
- `/drupal-best-practices-forms` → 2.7 Forms (Webform vs. Contact)
- `/drupal-best-practices-text-formats` → 2.8 Text formats and editors (CKEditor usage, allowed tags)
- `/drupal-best-practices-users` → 2.10 Users, roles & permissions (persona roles, permissions, dashboards)
- `/drupal-best-practices-theming` → 3 Theming & templates (theme naming, atomic design, twig/preprocess)

**Usage tip:** Pair commands with file or module names (e.g., `/drupal-best-practices-views blog_listing`) to request targeted checks or generation.
